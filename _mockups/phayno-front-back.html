Phayno (Chasuble) — Front & Back Display Options

Exploring approaches to show both front and back views of the vestment

Approach 1: Side-by-Side (Horizontal)

Display front and back images next to each other in a two-column layout.

👔

🔄

Back

✓ Pros: Both views visible simultaneously. Easy comparison. No interaction needed. Familiar pattern (like product pages).
✗ Cons: Takes more horizontal space. May be cramped on mobile. Not as immersive as full-size.

Approach 2: Stacked with Toggle Buttons

Display one full-size image at a time with toggle buttons to switch between front and back.

👔
✓ Pros: Large, immersive view of each side. Full details visible. Mobile-friendly. Clean UI.
✗ Cons: Requires interaction to see both. User must click/tap to compare. Can't see both simultaneously.

Approach 3: Carousel with Slide Indicators

Horizontal swipe/slide carousel showing front, then back, with dot indicators showing position.

👔
✓ Pros: Familiar mobile pattern (matches carousel metaphor). Smooth transitions. Clear progress indicator.
✗ Cons: Similar to toggle (can't see both at once). Carousel behavior might feel redundant with existing carousel page.

Approach 4: Interactive Flip (Desktop) + Toggle (Mobile)

On desktop: hover or click to flip the image with animation. On mobile: toggle buttons to switch views.

👔

Hover to flip →

✓ Pros: Delightful interaction. Takes advantage of desktop capabilities. Clear metaphor (like a physical card). Mobile-appropriate fallback.
✗ Cons: Complex to implement. Hover doesn't work on mobile (needs fallback). Extra effort for questionable UX gain.
💡 Recommendation: Approach 2 (Stacked with Toggle Buttons) offers the best balance:
  • Immersive, full-size view of each angle (important for appreciating vestment details)
  • Mobile-friendly with minimal responsive complexity
  • Consistent with existing carousel interaction patterns
  • Minimal code complexity compared to flip animations
  • Can extend pattern to other multi-view items (Dhoopakutty chains, other vestments)
